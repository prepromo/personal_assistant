# Если WireGuard так и не заработал

## 1. Отделить «что уже работает»

У вас уже было: **handshake** в `wg show`, сервер отвечает, IP **95.81.112.200** верный. Значит проблема не в «вообще не тот сервер», а в **доставке полезного трафика** (часто MTU, IPv6, фаервол Windows, сеть провайдера по UDP).

## 2. Три рабочих выхода (выберите один)

### A) Не мучить себя с самосбором — платный VPN на месяц

**Mullvad / Proton / IVPN** и т.п.: поставил клиент → подключился → проверил сайт. Это **не поражение**, а экономия нервов, если нужен просто рабочий выход в интернет.

### B) Остаться на своём VPS, но не WireGuard — REALITY (TCP)

В репозитории уже есть **`server/install-xray-reality.sh`**. На многих сетях, где **UDP режут**, **TCP + REALITY** проходит, когда WG нет. Поднимаете по README, импортируете ссылку в **v2rayNG / Hiddify** на телефоне или ПК.

### C) Один «чистый» прогон WireGuard с нуля (если хотите именно WG)

1. **Новый тестовый VPS** в **другой стране** (дешёвый trial / €4) — исключить «особенности» текущего хостинга и сети.
2. Только **Ubuntu**, один скрипт: `setup-wireguard-two-clients.sh` или `install-wireguard.sh`.
3. Сначала **только телефон + LTE**, конфиг с **`MTU = 1280`**, **`AllowedIPs = 0.0.0.0/0`** без `::/0`.
4. На ПК — **импорт свежего conf**, фаервол: правила для интерфейса туннеля (см. README), скрипт **`client/windows/check-vpn-health.ps1`**.

Если на **новом** VPS с **телефона по LTE** заработает, а на старом нет — дело в **старом сервере/сети**. Если нигде не работает — смотреть **вашего провайдера и UDP**.

## 3. Кому отдать сервер «под ключ»

Если нужно именно «чтобы кто-то залез по SSH и настроил» — это **отдельный человек** (знакомый админ, фриланс, поддержка хостинга с платной админкой). Из этого чата **нельзя** зайти на ваш VPS и ввести пароль.

## 4. Минимальная проверка после любых правок

- Включён VPN → `check-vpn-health.ps1` (из `client/windows`).
- На сервере: `sudo wg show` — есть handshake и растёт **transfer** у вашего пира.

---

**Коротко:** либо **платный VPN** ради спокойствия, либо **REALITY на том же VPS**, либо **новый VPS + чистая установка + сначала телефон**. Самосбор WG при живом handshake, но без сайтов — почти всегда **MTU / IPv6 / Windows / UDP по сети**, а не «одна волшебная галочка в панели».
