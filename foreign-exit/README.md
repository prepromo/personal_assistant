# Отдельный VPN «выход в другую страну»

Это **не отдельный протокол**, а **второй WireGuard-сервер** на VPS в нужной стране (EU/US и т.д.). Клиент на ПК/телефоне — **второй туннель** в приложении WireGuard (имя файла/туннеля другое, например `nl.conf` / туннель `nl`).

## Что нужно

1. Арендовать **ещё один VPS** в нужной стране (любой провайдер).
2. На нём один раз выполнить установку из этого репозитория, как для основного сервера, например:

   ```bash
   sudo bash setup-wireguard-two-clients.sh
   # или install-wireguard.sh + wg-add-client
   ```

   В `Endpoint` указать **публичный IP этого нового VPS** (не российский).

3. Импортировать выданный `.conf` в WireGuard под **другим именем** туннеля, чтобы не пересекаться с первым.

4. Включать **либо** один туннель, **либо** другой (два full-tunnel сразу без ручной маршрутизации обычно не нужны).

## Проверка соединения с ПК (Windows)

Из каталога `client/windows` (или по полному пути):

```powershell
.\check-vpn-health.ps1 -InterfaceAlias nl
```

Без `-InterfaceAlias` скрипт попробует найти интерфейс WireGuard. Смотрит: DNS/HTTPS до `1.1.1.1`, внешний IP через HTTPS, опционально страну по ipinfo.

---

Панель DNS/доменов хостинга на **старый** VPS к «второй стране» не привязана — только **IP нового сервера** в `Endpoint`.
