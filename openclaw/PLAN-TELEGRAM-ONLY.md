# План v1: только Telegram-бот (OpenClaw)

Цель: **не тянуть** сейчас веб-MVP и личный MTProto (`telegram-user`) как обязательную часть продукта. Всё, что видит пользователь, — **диалог с ботом** из @BotFather, агент **OpenClaw + GigaChat** (как у вас в `openclaw/.env`).

## Что делает бот

1. **Принимает команды** в Telegram (текст, что умеет ваш агент в OpenClaw).
2. **Отвечает** через LLM (GigaChat через gpt2giga или напрямую — см. README).
3. **Напоминания «ответить»** — реализуются **внутри сценария агента** (напоминание в том же чате с ботом через отложенные сообщения / cron OpenClaw / skills), а не через отдельный `telegram-user` API.
4. **Задачи** — пока как **контекст диалога** и память агента (или файлы/skills OpenClaw); при росте — можно снова подключить HTTP к `telegram-user` как бэкенд.

## Что не нужно для этого плана

- Запуск **`telegram-user`** (`npm run dev` на 4050) — **опционально**, только если решите связать бота с REST.
- Открывать **http://127.0.0.1:4050** — не обязательно для «только бот».

## Как запустить только бота

1. Заполнить **`openclaw/.env`**: `TELEGRAM_BOT_TOKEN`, ключи GigaChat, при необходимости gpt2giga.
2. Один раз: `.\scripts\install-openclaw.ps1` и при необходимости `.\scripts\install-gpt2giga.ps1`.
3. Запуск:  
   `cd openclaw` → **`.\scripts\start-gateway.ps1`**
4. В Telegram написать **вашему боту** (тот же токен в `.env`).

Проверка: `openclaw gateway status`, `openclaw health --json` (см. основной README).

## Про 404 на localhost:4050

Это API **`telegram-user`**. Страница панели: **http://127.0.0.1:4050/mvp.html** (или после обновления — редирект с **http://127.0.0.1:4050/** на `/mvp.html`). Если API не запущен — соединение не откроется.

## Следующий шаг в коде (по желанию)

- Прописать в конфиге OpenClaw **системный промпт**: роль «помощник в мессенджерах, напоминания, задачи».
- Добавить **skill** или **tool** под напоминания, если OpenClaw это поддерживает в вашей версии.

Итог: **один процесс — gateway + Telegram**, без обязательного `telegram-user` для первого рабочего демо.
