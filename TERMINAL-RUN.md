# Запуск всего стека только из терминала (в т.ч. встроенный терминал Cursor / VS Code)

Этот файл описывает **все части проекта**, **что за что отвечает**, и **как поднять их командами в PowerShell**, не выходя из окна терминала внутри редактора. Без сводных таблиц — только по шагам и по смыслу.

---

## Что у вас вообще есть в репозитории

**Корень `VPN_service`** — монорепозиторий. Главные «живые» части для вашего сценария (бот + личный Telegram + LLM):

1. **`telegram-user/`** — Node.js: HTTP API (порт **4050** по умолчанию), веб-страницы кабинета, **продуктовый бот** Telegram (Bot API, токен `PRODUCT_BOT_TOKEN`). Это «мозг» продукта в смысле бизнес-логики и БД.

2. **PostgreSQL** — база данных. Локально поднимается через Docker из каталога `telegram-user` (скрипт `docker compose` внутри `start-local-stack.ps1`). Без БД API не сможет работать с аккаунтами и сообщениями.

3. **gpt2giga** (скрипты в **`openclaw/scripts/`**) — локальный прокси в стиле OpenAI API, обычно слушает **`http://127.0.0.1:8090/v1`**. Нужен, если в **`telegram-user/.env`** указаны `OPENAI_BASE_URL` на этот адрес и ключ (как в `.env.example`). Иначе NL и другие LLM-вызовы в `telegram-user` будут падать с `ECONNREFUSED`.

4. **OpenClaw gateway** — отдельный процесс из пакета OpenClaw (`openclaw/scripts/start-gateway.ps1`). В полном стеке он часто запускается **вместе** с gpt2giga и настраивает окружение. Для самого **`telegram-user`** LLM может идти **напрямую в gpt2giga**, если в `telegram-user/.env` **не** заданы одновременно `OPENCLAW_GATEWAY_URL` и `OPENCLAW_GATEWAY_TOKEN` — тогда gateway не обязателен для ответов бота, но скрипт «полного стека» всё равно может его стартовать (см. ниже про режимы).

5. **`telegram-user/connector/`** — Python: **воркер** (`worker.py`) с Pyrogram. Он подключается к вашему **личному** Telegram (MTProto), синхронизирует диалоги и сообщения в БД и отправляет исходящие из очереди `TgPendingSend`. Без воркера личный аккаунт «не оживает» в продукте, даже если API запущен.

6. **Скрипты запуска в корне** — **`StartCode.ps1`** (основной вход) и **`Start.ps1`** (он только вызывает `StartCode.ps1` с теми же параметрами). Оба запускаются **из терминала** обычной командой `.\StartCode.ps1` или `.\Start.ps1` — это не отдельная программа вне терминала, это PowerShell-скрипты в репозитории.

7. **`scripts/start-tg-stack.ps1`** — низкоуровневый запуск «связки» из корня: либо отдельные окна PowerShell (**`-DesktopWindows`**), либо один терминал через **`scripts/tg-stack.mjs`** (режим без `-DesktopWindows`).

---

## Важно: «только терминал внутри Cursor» vs «новые окна Windows»

Режим **`StartCode.ps1 -Mode FullDesktop`** (пункт **1** в меню) **намеренно открывает дополнительные окна** PowerShell для gateway, API и воркера. Это удобно на рабочем столе, но **не** то, когда вы хотите, чтобы **всё осталось в одной вкладке терминала редактора**.

Чтобы **весь вывод шёл в одном терминале** (встроенном в Cursor), используйте связку:

1. Сначала **`Deps`** — поднимает Postgres, миграции Prisma и gpt2giga.
2. Потом **`FullHere`** — в **том же** терминале через Node запускает gateway, API с задержкой и воркер с задержкой (логи с префиксами `[gateway]`, `[api]`, `[worker]`).

Перед первым **`FullHere`** в корне репозитория один раз выполните **`npm install`**, чтобы появился пакет `concurrently` для `scripts/tg-stack.mjs`.

## Рекомендуемый полный запуск в одном терминале редактора

Откройте терминал PowerShell, перейдите в корень репозитория (подставьте свой путь, если он другой):

```powershell
Set-Location D:\Python\Github\VPN_service
```

Один раз установите зависимости корня (для `FullHere`):

```powershell
npm install
```

Поднимите базу и LLM-прокси:

```powershell
.\StartCode.ps1 -Mode Deps
```

Затем в том же окне запустите полную связку в текущем терминале:

```powershell
.\StartCode.ps1 -Mode FullHere
```

Остановка: сфокусируйтесь на этом терминале и нажмите **Ctrl+C** — завершится процесс Node, дочерние шаги останавливаются вместе с ним (как правило; при зависании отдельного дочернего PowerShell может потребоваться закрыть вкладку или завершить процесс вручную).

Проверка: в браузере откройте **`http://127.0.0.1:4050/health`** — в ответе должно быть `"ok": true`.

### Частая ошибка: «Имя StartCode.ps1 не распознано»

Скрипт **`StartCode.ps1` лежит только в корне репозитория** (`VPN_service`), рядом с папкой `telegram-user`. Если после `Deps` вы перешли в каталог `telegram-user` (так показывает приглашение `PS ...\telegram-user>`), команда **`.\StartCode.ps1`** ищет файл **внутри** `telegram-user` — его там нет, PowerShell выдаёт ошибку.

**Что сделать:** вернуться в корень и снова вызвать скрипт:

```powershell
Set-Location D:\Python\Github\VPN_service
.\StartCode.ps1 -Mode FullHere
```

Либо из `telegram-user` без смены каталога:

```powershell
& D:\Python\Github\VPN_service\StartCode.ps1 -Mode FullHere
```

(подставьте свой путь к корню, если он другой).

## Запуск через меню (тоже только терминал)

Если вызывать без параметра **`Mode`**:

```powershell
.\StartCode.ps1
```

или:

```powershell
.\Start.ps1
```

откроется **текстовое меню** в том же терминале (цифры 1–5). Пункт **2** соответствует **`FullHere`**, пункт **3** — **`Deps`**. То есть **`Start` и `StartCode` уже работают «внутри программы»** — это обычный интерактив PowerShell в панели терминала Cursor.

---

## Если нужны только отдельные куски (всё ещё из терминала)

**Только API и продуктовый бот в текущем терминале** (без нового окна): не используйте для этого `telegram-user/scripts/start-dev.ps1` без правки — он по умолчанию открывает **новое** окно. Вместо этого:

```powershell
Set-Location D:\Python\Github\VPN_service\telegram-user
npm install
npx prisma migrate deploy
npm run dev
```

**Только воркер** в текущем терминале (после настроенного venv в `connector`):

```powershell
Set-Location D:\Python\Github\VPN_service
.\telegram-user\scripts\start-worker.ps1
```

без флага **`-NewWindow`**. С флагом `-NewWindow` снова откроется отдельное окно Windows.

**Только OpenClaw gateway** (блокирует терминал до Ctrl+C):

```powershell
Set-Location D:\Python\Github\VPN_service\openclaw
.\scripts\start-gateway.ps1
```

## Переменные окружения (кратко, без таблиц)

Перед любым заполните **`telegram-user/.env`** по образцу **`telegram-user/.env.example`**: как минимум **`DATABASE_URL`**, секрет коннектора, при необходимости **`PRODUCT_BOT_TOKEN`**, и строки для LLM (часто **`OPENAI_BASE_URL=http://127.0.0.1:8090/v1`** и ключ). Для сценария с gateway заполните **`openclaw/.env`** по примеру в каталоге `openclaw`. Без Docker Desktop локальный Postgres из `Deps` не поднимется.

---

## Итог

Чтобы **поднять всё полностью одной логикой и остаться в одном терминале IDE**, используйте **`Deps`**, затем **`FullHere`**, из корня репозитория, с предварительным **`npm install`** в корне. Скрипты **`Start.ps1`** и **`StartCode.ps1`** как раз для этого и предназначены — они выполняются там же, где вы ввели команду в терминале; отдельное «подключение» не требуется. Режим с отдельными окнами — это только **`FullDesktop`** (пункт меню **1**).
